沒有採購申請

請先在「連接設定」填入 Google Sheets API 網址，然後點擊同步。

📋 連接你的採購 Apps Script

把下方第二步驟拿到的 WEBAPP_URL 貼入下方，網站就能讀取所有採購申請，且完全不影響你原有的 Email 通知流程。

💡 你的 WEBAPP_URL 在 Apps Script 的 CONFIG 裡，格式為
https://script.google.com/macros/s/…/exec
未設定

🆕 建立一個全新、獨立的讀取用 Apps Script

這個全新的 Apps Script 跟你原本處理 Email 審批流程的腳本完全分開、互不影響，它唯一的功能就是讀取 Google Sheet 資料給這個網站用。

步驟：
1. 打開 script.google.com，點「新增專案」
2. 打開你原本連接表單的那份 Google Sheet，從網址列複製 Sheet ID（/d/ 和 /edit 之間那一串）
3. 點下方「複製參考程式碼」，貼到新專案裡，把 SHEET_ID 換成剛複製的那串
4. 點 部署 → 新增部署，類型選「網頁應用程式」，執行身分 = 我，存取權限 = 所有人
5. 部署後會拿到一個新的 .../exec 網址，貼到下方欄位
✅ 完全是另一個專案，不會動到你原本 Email 審批流程的任何程式碼
✅ 網站只「讀取」資料，不會修改任何 Sheets 內容
✅ 到貨日期、狀態備註儲存在瀏覽器本地，不影響 Sheets

如果測試連線顯示「連線逾時」，最快的排查方式是直接把新網址（含 ?action=getRequests）貼到瀏覽器網址列開啟——能看到 JSON 資料表示腳本本身沒問題；看到錯誤訊息，通常是 SHEET_ID 貼錯或還沒授權存取該表單。
